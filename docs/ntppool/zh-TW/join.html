我該如何加入 pool.ntp.org ？

首先：感謝你對此感到有興趣，想知道更多。 共用計畫的使用量持續顯著地在增長，而唯一能讓我們便利地持續提供服務的方式便是增加更多參與的伺服器。

你的主機必須要有固定的 IP 位址和持續的網際網路連結。 擁有固定不變或極少變動（例如一年或更長時間內只變動一次）的 IP 位址對於架設時間伺服器來說十分重要。 架設需要的頻寬則是相對較低的。 每個用戶只會在每幾分鐘到每 20 分鐘之內傳送幾個 UDP 封包。

目前來說大多數伺服器每秒會收到約 5-15 個 NTP 封包，有時一天之內會有數次每秒收到 60-120 個封包的尖峰期。 這約略等同於每秒 10-15Kbit 的流量，尖峰期的流量則可達每秒 50-120Kbit 之多。 本計畫持續地在徵求更多的時間伺服器，所以每個伺服器的負載應該不至於會有太過戲劇性的增長。 白話來說，至少應該要具備 384-512Kbit 的頻寬（上傳及下載）。

加入的伺服器不應設定使用 pool.ntp.org 作為其上游時間伺服器，而該手動設定一些已知的良好伺服器 （該些伺服器可以從共用計畫的伺服器當中選取出來。 這樣要求的意義在於時間來源應該要是恆定的，而非每次伺服器重新啟動時隨機選用。 這樣有助於維持良好的時間服務品質。） 注意本計畫並不要求你的伺服器必須要是第一層或第二層時間伺服器 - 因為本計畫的目標最主要在於分散負載，是以即便是第三層或第四層時間伺服器，也沒有理由拒絕它的加入。

我們也有提供建議的設定供有意加入的伺服器管理者參考。

最後要再次強調，加入共用計畫是一個長期性的承諾。 如果狀況有變，我們樂意協助將你的伺服器移出共用計畫，但受限於 NTP 用戶端本身的運作方式，可能需要數周、數月甚至數年的時間流量才會完全消失。

如果這些都了解了，請登入伺服器管理頁面並登錄你的伺服器。 若操作上遇到困難，可以來信至 ask@develooper.com 尋求協助。

我們也邀請你訂閱共用計畫網誌 feed 以及共用計畫的 mailing list 。

如果可以的話（這並不是必要的），請協助將流向你的時間伺服器的 port 80 之網頁要求轉導向至本計畫的官方網站 http://www.pool.ntp.org 。 如果你有使用 Apache ，可以這樣設定：

[% PROCESS tpl/join/virtual_host_example.html %]

當然，這還是在你的主機有架設網頁伺服器的情況下才需要如此。 本計畫的官方頁面都有以 'www' 字樣開頭，但總有懶人直接輸入 pool.ntp.org 於是發現被導到了某個隨機的網站。

加入共用計畫的伺服器都會持續受到監測以確認其準確度和可用性。 你可以使用網頁前端或管理頁面來監控你的伺服器的效能數值。
